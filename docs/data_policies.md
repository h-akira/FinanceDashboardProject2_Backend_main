# データポリシー

## 概要

本ドキュメントは、外部データソースからのデータ取得・加工・結合に関するポリシーを定める。テーブル設計については [database_design.md](./database_design.md) を参照。

---

## 共通ポリシー

### データの境界

各データソースには「固定データ」と「動的データ」の境界日を設ける。

| 区分 | 保持先 | 取得タイミング |
|------|--------|--------------|
| 固定データ（境界日以前） | DynamoDB | 手動スクリプト（`bin/`）で事前登録 |
| 動的データ（境界日以降） | レスポンスに動的結合 | Lambda がリクエスト時に取得 |

- 動的データは DynamoDB に書き込まない
- DynamoDB の格納済みデータと日付が重複する動的データは除外する

### サンプリングルール

データソースごとにサンプリング方式を定義する。以下は現時点での標準方式である。

#### 半月単位サンプリング

1. 各月の **1 日** と **15 日** を基準日とする
2. 基準日が営業日でない場合（土日祝でデータが存在しない場合）、**直後の最初の営業日**のデータを採用する
3. 前処理として forward-fill（前方補完）を行い、欠損値を除去した上でサンプリングする

動的データでは上記に加え、**最新の営業日のデータ**も含める（例: 3 月 20 日にリクエスト → 3 月 1 日・15 日・20 日の 3 データポイント）。

## データソース別定義

### FRED（Federal Reserve Economic Data）

| 項目 | 値 |
|------|-----|
| 取得方法 | pandas_datareader |
| 元データ粒度 | 営業日単位（土日祝を除く） |
| サンプリング | 半月単位サンプリング |
| 境界日 | 2025-12-31 |
| 登録スクリプト | `bin/load_interest_rate.py` |

#### データ系列

| データ系列名 | FRED シリーズ ID | 説明 |
|-------------|-----------------|------|
| `target_rate` | DFEDTAR（〜2008）/ DFEDTARU（2008〜） | FF 金利誘導目標（上限） |
| `dgs10` | DGS10 | 米国 10 年国債利回り |

## データソース追加手順

新しいデータソースを追加する場合:

1. 本ドキュメントに「データソース別定義」セクションを追記する（取得方法・粒度・サンプリング・境界日・データ系列）
2. [database_design.md](./database_design.md) に PK パターンとアクセスパターンを追記する
3. `bin/` に登録スクリプトを作成する
4. サービス層に動的取得・結合ロジックを追加する
